基础

vue中箭头函数

所有被vue管理的函数最好都写成普通函数

不被vue管理的 如定时器，请求等， 最好使用箭头函数，（这样this的指向才是VM或者组件的实例对象）

mvvm

  • M 模型（Model）： 对应data中的数据
  • V: 视图（View）： 模板
  • VM： 视图模型（）： Vue实例对象

Object.defineproperty方法

可以用来给对象添加属性

  • 数据劫持 数据代理 计算属性 都有使用到该方法

    通过该方法添加的属性不可以枚举（遍历），如需遍历在第三个参数的配置项中添加 enumerable: true

参数：1、要添加的目标对像

​ 2、添加的属性 名

​ 3、配置项

Object.defineProperty(person,'age', {
    value: 18, // 值
    enumerable: true, //控制属性是否可以枚举
    writable: true, // 值是否可以被修改， 默认为false
    configurable： true, // 属性是否可以被删除， 默认为false
    // 访问时触发get方法 返回值为age的值
    get() {}
    // 修改时触发， 且会收到修改的具体值
    set(value) {}
})

数据代理

通过一个对象代理对另一个对象的操作 读或写

// 简易数据代理
let obj = {x: 1}
let obj2 = {y: 2}

Object.defineProperty(obj2, 'x', {
  enumerable: true,
  get() {
    return obj.x
  },
  set(value) {
    obj.x = value
  }
})

obj2.x = 3;

console.log(obj2);

计算属性

好处 缓存 计算属性中this指向vue的实例

两种写法 例如计算sum属性 

 // 复杂写法——使用配置选项写法 {} （考虑修改属性的时候）
 computed: {
     sum: {
         get() {},
         set() {}
     }
 }

但大多数的情况下是不考虑修改的情况的——简写

// 简写时使用函数写法 可以解为 sum() === get()
computed: {
    sum() {
        // 具体的操作
    } 
}

监听器 watch

监听数据的改变，内置了很多的 配置选项 如监听 布尔变量flag

watch: {
    flag: {
        // 内置handler函数 当flag属性发生变化时 就会自动调用
        // 有两个参数 第一个新的值 第二个旧的值
        handler(newValue, oldValue) {
            console.log(newValue)
            console.log(oldValue)
        },
        // 初始化时 立即执行配置
        immediate:true
    }
}

另一种写法 通过 实例.$watch('监听的属性', {配置项})

// 监听的 属性要加 “
vm.$watch('flag', {
    // 内置handler函数 当flag属性发生变化时 就会自动调用
    // 有两个参数 第一个新的值 第二个旧的值
    handler(newValue, oldValue) {
        console.log(newValue)
        console.log(oldValue)
    },
    // 初始化时 立即执行配置
    immediate:true
})

多级结构监视

watch: {
    // a.b: {} 错误写法
    // 正确写法 通过 . 调用是不能使用简写 要写成原始写法加上引号
    'a.b': {}
}

深度监视 deep

// 当监视的属性是一个多级的对象那个，想监听每个元素时 添加配置项
obj： {
    a: 1，
    b: 2，
    ......
}
watch: {
    // obj: {} 直接写是监听不了的 需要开启深度监听
    obj: {
        // deep 开启深度监听
        deep: true,
        handler() {}
    }
}

计算属性对比监听器

1、computed能完成的功能watch都可以完成
2、watch能完成的，computed不一定能完成， 如watch可以完成异步
3、watch开销相对较大，一般用在请求 异步时
4、计算属性具有缓存 性能更好

key

// key使用index时的问题
逆序添加时的问题（破坏顺序时）
原因：key是唯一标识，在用户操作dom时，vue使用的是 diff算法对比虚拟dom是否改变，以key为依据。只有发生改变时才会更新视图。
使用key时，在头部插入数据，原先的元素的key会改变，在diff比对时就会出现问题，导致后续元素的部分内容都要更新，导致页面错误

自定义指令

使用配置对象： directives

// 1、函数写法
<div v-big=n></div>
// big函数调用时机， 1、指令和元素成功绑定时 2、指令所在的模板被重新解析时
directives: {
// 两个参数， 1、真实的dom元素， 2、绑定
    big(element, binding) {
        element.inberText = binding.value * 10
    }
}

// 2、 配置对象写法
directives: {
    big: {
        // 指令和元素成功绑定时调用
        bind(element, binding) {}
        // 指令所在元素被插入页面时调用
        inserted(element, binding) {}
        // 指令所在的模板被重新解析时调用
        update(element, binding) {}
    }
}

全局事件总线

可实现任意组件间通信

原理： 定义一个总线qj，能够被说有组件访问，且具有$on $emit方法

    在组件上自定义事件，事件内总线qj通过$on绑定事件，其他组件内通过总线$emit触发传参（就可以实现任意组件通信）
    (this.$bus.$on('事件名')) // 挂载
    (this.$bus.$emit('事件名','数据')) // 触发

总线qj： (条件) 
    1、全部组件可访问： 通过在入口文件上 在Vue。prototype挂载对象实现
    2、具有$on $emit: 
    
具体实现: 
    new Vue({
        el: '#app',
        render: h => h(app),
        beforeCreate() {
            // 安装全局事件总线
            Vue.prototype.$bus = this 
        }
    })

    (this.$bus.$on('事件名','回调函数')) // 挂载
    (this.$bus.$emit('事件名','数据')) // 触发

注： 使用完事件后最好将全局的事件进行解绑
