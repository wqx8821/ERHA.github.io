路由是什么

​ 对于前端来说，路由概念的出现是伴随着spa出现的；在spa出现之前，页面的跳转(导航)都是通过服务端控制的，并且跳转存在一个明显白屏跳转过程；spa出现后，为了更好的体验，就没有再让服务端控制跳转了，于是前端路由出现了，前端可以自由控制组件的渲染，来模拟页面跳转

  • 服务端路由根据url分配对应处理程序，返回页面、接口返回
  • 前端路由是通过js根据url返回对对应组件

如何实现路由

​ 路由映射表： 要实现路由，需要一个路由映射表，前端映射表是 url 与 组建的关系

​ 匹配器： 有了映射表，只是有了对应的映射关系，并不能通过url访问对应的组件，因此还需要一个匹配器

​ 历史记录栈：对于浏览器的前进后退的功能，需要一种栈的数据结构进行存储url，浏览器有封装好的接口，直接调用即可 window.history

​ 用户操作流程：访问X路径————匹配器去映射表查询返回对应的组件————将历史记录推入 stack中————前进后退时（从栈中取出url 匹配器去 映射表查询 返回对应的组件）

vue路由组成

路由对象

  • Route 表示当前激活路由的状态信息，包含了当前 URL 解析得到的信息，还有 URL 匹配到的路由记录们 (route records)。
  • 通过this.$route访问到的就是Route对象
  • 路由对象是不可变的，每次导航都会产生新的对象

位置(Location)

  • 它并不是window.location的引用，vue-router在内部定义了一个Location，是一个用来描述目标位置的对象；
  • $router.push/replace router-link的to接收的就是Location对象
  • vue-router内部可以将一个url string转换成Location对象，所以确切的说$router.push/replace、router-link的to接收的都是一个RawLocation对象
  • RawLocation对象是String和Location的联合类型

路由组件(RouteComponent)

  • 当路由成功匹配时，就需要在router-view渲染一个组件，这个需要被渲染的组件就是路由组件
  • RouteConfig中component、components中定义的vue组件就是路由组件
  • 路由组件的特殊性
    • 拥有只在路由组件中生效的守卫(beforeRouteEnter 、beforeRouteUpdate、beforeRouteLeave)
    • 你是否跟我一样，曾经在组件中调用beforeRouteEnter发现没有生效，那是因为这个守卫只能在路由组件中被调用，在所有非路由组件中都不会被调用，包括路由组件的后代组件；你如果想在路由组件中实现beforeRouteEnter类似守卫监听效果，可以通过watch $route来手动判断

简单实现

​ 实现基础的 插件（只实现了 hash模式） 以及两个内置组件

​ 路由历 史 缓存 拦截器 以及N多配置项 还需多努力 （简易版并未实现）

// 实现一个插件(实质是一个函数或对象) 以及 两个组件

let Vue; // 保存vue的构造函数

class VueRouter {
    constructor(options) {
        // 缓存配置项
        this.$options = options
        
        // 定义一个变量用来保持当前的路由的url
        // current需要是响应式的 不然current变化时 不会触发下面渲染render函数执行
        // 通过Vue内置的工具函数 difineReactive 将 current设置为响应式
        const initial = window.location.hash.slice(1) || '/'
        Vue.util.defineReactive(this, 'current', initial)
        
        // 监听hash 的变化，变化就保存 
        window.addEventListener("hashchange", () => {
            this.current = window.location.hash.slice(1) || '/'
            
            // console.log(this.current);
        })
    }
}


VueRouter.install = (_Vue) => {
    Vue = _Vue // 保存Vue 防止打包的时候将 Vue也打包进来

    // 挂载 $router 属性
    // 全局混入（延迟以下逻辑到router创建完【使用时 new Router】毕且附加到选项时【main.js 作为配置项传入】才执行）
    Vue.mixin({
        // 这个钩子在每个组件被创建的时候都会被调用
        beforeCreate() {
            // 根据配置项的参数 判断是否为根实例
            if (this.$options.router) {
                // 根实例存在就 挂载到Vue的原型上
                Vue.prototype.$router = this.$options.router
            }
        }
    })

    // 实现两个组件 （为什么可以直接使用 因为是全局的，因此在install时注册即可）

    Vue.component("router-link", {
        props: {
            to: {
                type: String,
                required: true
            }
        },
        // 渲染函数的方式来写
        render(h) {
            return h('a', {
                    attrs: { // 访问组件属性，将 to的内容给 a 的herf属性
                        herf: `#${this.to}`
                    }
                },
                this.$slots.default // 通过插槽获得 router-link 的文本内容
            )
        }
    })
    // 根据link的地址 去路由表中匹配对应的路由进行渲染
    Vue.component("router-view", {
        // 渲染函数的方式来写
        render(h) {
            let component = null
            
            // 获取当前的路由路径 (前边 全局混入了 $router 所以这里可以直接获取)
            const current = this.$router.current
            
            // 构造函数中 缓存了optons 根据里面的 routes 配置型查找路由是否存在
            const route = this.$router.$options.routes.find(res => res.path === current)
            // 若存在就说明找到了对应的路由 进行渲染
            if(route) {
                // route.component 在用户的路由配置中 对应的的组件保存下来 进行渲染
                component = route.component
            }
            return h(component)
        }
    })
}

// 导出
export default VueRouter;

简版Vuex实现

思路：Vuex是vue推荐周边的一款状态管理插件，实质是通过Mixin的方式为每一个组件注入store对象，由于每个组件的store都指向同一个store对象，所以store是全局 的，实现了状态共享， 类似一个事件总线

插件：向外暴露install方法，vue通过use使用

测试用例——store/index.js

在store文件夹下建立vuex.js文件，并在index.js使用

import Vue from 'vue';
import Vuex from './vuex.js';

Vue.use(Vuex);

export default new Vuex.Store({
  state: {
      count: 0
  },
  getters: {
        doubleCounter(state) {
            return state.count++
        }
  },
  mutations: {
      add(state, num) {
          return state.count += num
      }
  },
  actions: {
      add({commit}) {
          setTimeout(() => {
              commit('add',111)
          }, 1000)
      }
  },
  modules: {
  },
});

简易版Vuex实现 store/vuex.js

在store文件夹下建立vuex.js文件，并在index.js使用

// 简易版vuex （插件 挂载$store）
/*
state要具有响应式
 实现了 $store state
（mutations）commit 
（actions）dispatch
*/

let Vue // 缓存vue
class Store {
    constructor(options) {
        // 借助Vue的data实现 将state数据转为响应式
        /* 第一种写法 实现用户通过 this.$store.state.XX 形式访问
        this.state = new Vue({
            data() {
                return options.state
            }
        })
        */
        // 实际上在源码中，不希望直接向外暴漏Vue的实例，因此有多包装了一层
        // 第二种写法 实现用户通过 this.$store.state.XX 形式访问
        this._vm = new Vue({
            data: {
                $$state: options.state
            }
        })

        // 创建_mutations 接收配置项中的mutations
        this._mutations = options.mutations
        // 创建_actions 接收配置项中的actions
        this._actions = options.actions
        
        // 创建getters, 遍历存储所有的用户定义的getters 的key 当用户访问时就执行对应的函数（要实现缓存）
        this.getters = {}
        options.getters && this.handleGetters(options.getters)
        
        // 保存this
        this.commit = this.commit.bind(this)
        this.dispatch = this.dispatch.bind(this)
    }
    
    // 向外暴露state 只能get 不能set
    get state() {
        return this._vm._data.$$state
    }
    set state(v) {
        // 如果外界通过this.$store.state.XX 修改 就抛出一个错误
        console.err('Please use replaceState to reset state')
    }
    
    // 实现getters 这里还有点小问题
    handleGetters(getters) {
        Object.keys(getters).map(key => {
            Object.defineProperty(this.getters, key, {
                get: () => getters[key](this.state)
            })
        })
    }
    
    // 实现commit方法 有两个参数 
    // 第一个type是_mutations中对应的函数名称
    // 第二个payload参数为触发时传递的参数
    commit(type, payload) {
        // 获得配置型传递的_mutations对应的函数
        const entry = this._mutations[type]
        // 如果不存在这样的函数，就抛出一个错误
        if (!entry) {
            throw new Error('unknown mutation type')
        } else {
            // 存在就将state传递进函数, 并传递用户传递的第二个参数 并执行
            entry(this.state, payload)
        }
    }
    
    // 实现dispatch方法
    dispatch(type, payload) {
        // 获得配置型传递的_actions对应的函数
        const entry = this._actions[type]
        // 如果不存在这样的函数，就抛出一个错误
        if (!entry) {
            throw new Error('unknown mutation type')
        } else {
            // 存在就将state传递进函数, 并传递用户传递的第二个参数 并执行
            // 这里this指向会有问题(多组件嵌套，定时器异步都有可能造成) 使用bind保存this， 具体实现在上面的store类中
            entry(this, payload)
        }
    }
}

// 声明 install方法
const install = (_Vue) => {
    Vue = _Vue // 缓存vue

    // 全局混入
    Vue.mixin({
        beforeCreate() {
            // store在main方法中作为options传入（new Vue时）    // 即： 在options中存在store时 将store设为全局
            if (this.$options.store) {
                // 将store设为全局 挂载到vue原型
                Vue.prototype.$store = this.$options.store
            }
        }
    })
}

// 导出
export default {
    Store,
    install
}

补充_ Vue.use原理

软文：https://juejin.cn/post/6901224957959733261

Vue.use(plugin)

参数：

  • （Object | Function, []?） plugin 接受一个 obj 或者一个 fn , 第二个参数可选的一个数组

用法：

  • 安装Vue.js插件，如果插件是一个对象，必须提供install方法
  • 如果插件是一个函数，他会被视为install方法
  • ==install方法被调用时，会将Vue(构造函数)作为参数传入==
  • ==install方法 需要 在 new Vue之前被调用==
  • 因为传入Vue并在new 之前调用 （可以做很多事情，注册全局组件，注册全局指令，在Vue原型上挂载方法等） 提高Vue扩展性

原理流程：use的流程就是：

  • 先验证传进来的插件是否注册，存在即返回保证只注册一次。

  • 没注册的插件来判断插件类型。如果是对象必须有install方法，如果是函数直接插入。（记得改变this指向哦~）

  • 最后添加到已注册的插件列表里。

  • // Vue.use源码解析 位置initGlobalAPI 中 initUse
function initUse (Vue) {
  Vue.use = function (plugin) {
    // 定义查看已经被注册的组件，installedPlugins
    var installedPlugins = (this._installedPlugins || (this._installedPlugins = []));
    // 如果传入的组件已经被注册 直接返回实例
    if (installedPlugins.indexOf(plugin) > -1) {
      return this
    }
    
    // additional parameters（获取附加参数）
    var args = toArray(arguments, 1);
    // 向头部添加this
    args.unshift(this);
    // 判断入参是否有install方法
    if (typeof plugin.install === 'function') {
      // 有执行，动态改this为plugin
      plugin.install.apply(plugin, args);
    } else if (typeof plugin === 'function') {
      // 插件本身就是fn，动态改this为null
      plugin.apply(null, args);
    }
    // 已注册插件列表添加插件
    installedPlugins.push(plugin);
    return this
  };
}
