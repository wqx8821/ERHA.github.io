== 和===的区别

== 与 ===是不同的,一个是判断值是否相等,一个是判断值及类型是否完全相等

== 比较时会进行强制转换 ： 转换规则如下

null == undefined 相等 但与其他值都不想等
对象 == 字符串 ， 对象会toString（）变为字符串
NaN == NaN 不相等
其他的都会转换为数字

变量提升机制（预编译）

全局预编译

  1. 创建GO对象
  2. 找声明变量 将函数名作为GO对象的属性名 值为undefined
  3. 寻找函数声明 将值赋予函数体

函数预编译

  1. 创建AO对象
  2. 找 形参和变量声明 将变量和形参名 当作 AO对象的属性 值为undefined
  3. 实参形参统一
  4. 在函数体里寻找 函数声明 将值赋予函数体

作用域和作用域链

作用域： AO(函数作用域)　GO(全局作用域)

作用域链: AO, GO，　AO和GO的集合

​ 作用域都有上下级关系，上下级关系确定函数在哪个作用域下创建，变量取值都会在当前作用域中查找，如果没有查到就会像上级作用域查找，直到查到全局作用域，这个查找的过程叫做作用域链

扩展：nodejs中的作用域

为什么需要闭包

​ ==闭包的两大作用 ： 保护/保存==

​ 由于垃圾回收机制 ，局部变量无法共享和长久的保存，全局变量又可能造成变量污染，于是我们希望出现一种机制可以长久的保存局部变量，又不会造成变量污染，

​ 闭包就能解决这样的问题(闭包出现的 原因)，能够长久的保存局部变量在内存，而不会造成全居污染（闭包，函数内返回的是一个函数，并且该函数对局部变量进行的引用，并在外部被赋为全局变量==不会被回收的原因==）

​ 不必要的闭包会增加内存的消耗

防抖节流

​ 防抖： 当持续触发事件，一定时间内没有再触发事件，事件才会执行一次，如果设定的时间来到之前 又一次触发事件 就重新进行延时；例：

  • 如果在200ms内没有再次触发滚动事件，那么就执行函数
  • 如果在200ms内再次触发滚动事件，那么当前的计时取消，重新开始计时

效果：如果短时间内大量触发同一事件，只会执行一次函数。

// 防抖类似王者荣耀回城 打断重新计时

function                 () {
  console.log('show');
}

function debounce (func, time) {
  let timer = null;
  return function () {
    if(timer) {
      clearTimeout(timer)
    }
    timer = setTimeout( () => {
      func.call(this, arguments)
    }, time)
  }
}

let debounceShowLog = debounce(showLog, 3000);
debounceShowLog()

​ 节流：在规定时间内函数不会触发，必须等够时间才能触发

// 类似于王者英雄技能，释放过后必须冷却结束才可以再次触发

function canRun (func, time) {
  let flag = true;
  return function () {
    if (!flag) {
      return;
    }
    flag = false;

    setTimeout(() => {
      func.call(this, arguments);

      flag = true
    }, time)
  }
}

let canRunShowLog = canRun(showLog, 1000);
canRunShowLog()

浅拷贝深拷贝

// 浅拷贝与赋值的区别
赋值：当我们把一个对象赋值给一个新的变量，赋的是该对象在栈中的地址，而不是栈中的数据。两个对象指向同一个存储空间。
浅拷贝： 重新堆中创建内存，拷贝前后对象的`基本数据类型`互不影响，但对象若有深层次的（`引用数据类型`）拷贝的仍然是地址，由于共享同一块内存会互相影响。
深拷贝：在堆中重新开辟一个区域存放对象，对对象的更深层次的对象进行`递归拷贝`，完成后两个对象互不影响。
// 浅拷贝
在多维数据处理拷贝中，浅拷贝只为第一层开辟了新的内存空间，拷贝后的对象操作更深层次的数据时会改变原数据。因此需要进行深拷贝
原始数据(三层数据) obj = {
    name： "lcy",
    NAN: [1, 2, 3, 4],
    dx: {
        age: 18
    }
}
// 简单浅拷贝实现 
// let obj2 = {...obj}
// Object.assgin() 
// Array.prototype.concat() 
// Array.peototype.slice()
// es5方法
let obj2 = {};
for(let key in obj) {
    if(!obj.hasOwnProperty(key)) break;
    obj2[key] = obj[key];
}

// 深拷贝 
方法一：JSON.parse()将字符串改变为对象 JSON.stringify()将对象转变为字符串
let obj2 = JSON.parse(JSON.stringify(obj))
//问题： 原始数据内若有函数，日期对象，正则表达式会出现问题，无法拷贝copyObj对象原型链上的属性和方法，当数据的层次很深，会栈溢出，函数和正则表达式会替换为空对象，日期对象会变为字符串（理想的是变为标准日期对象）

方法二：因此采用递归的方法更为合适
function deepClone(obj) {
    // 过滤特殊情况
    if (obj === null) return null;
    if(typeof obj != 'object') return obj;
    if(obj instanceof RegExp) {
        return new RegExp(obj);
    }
    if(obj instanceof Date) {
        return new Date(obj);
    }
    // 不直接创建对象的目的：克隆的结果和之前的保存相同的所属类
    let newobj = new obj.constructor;
    for(let key in obj) {
        if(!hasOwnproperty(obj)) {
            newobj[key] = deepClone(obj[key])
        }
    }
    return newobj;
}
